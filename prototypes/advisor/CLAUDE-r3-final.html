R3 Final — Changes from R2-E:
1. Label size: 9px → 10px for WCAG readability compliance.
2. Standardized abbreviation rules for mod names (see table below). Same rules apply on both desktop and mobile — the gallery layout naturally handles wrapping at any width.
3. "Mod Components ON" variant shown to demonstrate Rare-tier mod icon colors (blue) vs Uncommon defaults (green).
4. "No attachment guide" state shown for weapons like Hairpin (simple italic text, no gallery row).
5. 4-slot weapon stress test demonstrated with Vulcano (all 4 icons fit comfortably).
6. Rarity badge moved inline with weapon name — rarity is part of weapon identity, not metadata. Frees the meta row.
7. "Build" renamed to "Customize" — communicates that mods come pre-loaded and you're going to tweak, not start from scratch. Now the sole element in the meta row for better visual weight.
8. Build button border + padding improved — reads as a button, not a label. Better mobile tap target.

Abbreviation rules for mod labels:
• "Extended" → "Ext."
• "Medium" → "Med."
• "Light" → "Lt."
• "Shotgun" → "SG"
• "Compensator" → "Comp."
• "Vertical" → "Vert."
• "Magazine" → "Mag"
• "Muzzle Brake" → stays as-is (recognizable, wraps naturally to 2 lines)
• "Stable Stock" → stays as-is
• "Angled Grip" → stays as-is
• Tier number always shown (I, II, III)

These abbreviations apply universally (not mobile-only) because the gallery's flex wrapping handles text length gracefully. No responsive-only shortening needed.
Engineering Implementation Notes

1. Placeholder icons → real attachment thumbnails
The rarity-colored squares (■) in this prototype are placeholders only. In production, replace each square with the actual mod's thumbnail image from the Arc Raiders wiki. Follow the same pattern used for weapon images: add a MOD_IMAGES lookup in src/data/constants.ts (keyed by mod family name or ID) that maps to wiki thumbnail URLs. The 36px icon square becomes a 36×36 <img> with object-fit: contain, and the rarity-colored border stays as a 2px solid border on the container (using the same RARITY_COLORS map already in constants).

2. WeaponBlock.tsx restructuring
Current layout has rarity badge + Build button on the same row below the weapon name. New layout:
• Name row: weapon name (16px bold) + rarity badge inline (same row, align-items: baseline)
• Meta row: "Customize →" button as the sole element (with margin-left: auto)
• Mod gallery row: new row below meta, only rendered when attachment build data is provided
The component should accept an optional attachmentBuild prop (array of equipped mods for recommended build). When absent, no gallery row renders (backwards compatible).

3. Mod gallery component
Extract as ModGallery.tsx in src/components/shared/ so it can be reused in the Builder if needed later. Props: array of mods (with slot type, mod name, rarity, image URL) + optional empty slot definitions. Layout: display: flex; justify-content: center; gap: 12px, each item uses flex: 1; max-width: 90px for even distribution regardless of 2/3/4 slots. Empty slots render as dashed-border containers with slot type label.

4. Abbreviation utility
Create a abbreviateModName(name: string): string utility (e.g., in src/utils/ or alongside constants). It should apply the substitution rules from the table above via a simple map lookup on individual words, preserving tier suffixes (I, II, III). Do not hardcode abbreviations per-instance in the component.

5. Crafting filter pills
Add to AdvisorFilterBar.tsx on the second row, to the right of weapon rarity filters. Use the pill-craft style class (green-tinted background, distinct from orange accent and colored rarity pills). Full labels: "Mech Components", "Mod Components", "Kinetic Converter", "Horizontal Grip". Toggle state uses checkmark/X prefix. Wire through useAdvisorFilters hook — filters exclude entire builds whose highest-rarity mod requires that crafting material (weakest-link rule), not individual mod slots.

6. "Customize →" deep link
The button should encode the recommended build's attachments into URL query params using the existing useBuildUrl system, then navigate to the Builder tab with those mods pre-loaded. Existing onOpenBuilder callback pipeline (WeaponBlock → PairingCard → AdvisorResults → AdvisorPage → App) already supports this — just extend the payload to include mod selections alongside the weapon ID.

7. "No attachment guide" state
Some weapons (e.g., Hairpin) have no recommended builds. When attachmentBuild is null/empty, render a simple italic "No attachment guide" text line instead of the mod gallery. No dashed placeholders, no empty gallery row — just a clean text fallback.
