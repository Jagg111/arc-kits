R4 Changes from R3:

1. Graduated visual hierarchy
Each build card gets a left-border accent colored by its highest-rarity mod (gold/pink/blue/green/gray). The numbered circle matches. At scan speed, players immediately see the "cost gradient" — expensive builds glow at the top, budget builds fade to neutral at the bottom. This leverages the looter-shooter rarity language players already understand.

2. Crafting requirement labels
Right-aligned text on each card's name row. Desktop: "Requires Legendaries" / "Requires Mod Components" / "Requires Mech Components" / "Base Materials". Mobile: shortened to "Legendaries" / "Mod Comps" / "Mech Comps" / "Base Materials". Color matches the rarity tier. This directly mirrors the Advisor's crafting filter vocabulary, creating system-wide consistency.

3. Weapon Intel sidebar
Desktop: always visible in sticky sidebar (same as R3) — no collapse toggle, no extra click. The content is compact enough to not compete with the build list. Mobile: collapsible accordion (collapsed by default) since vertical space is at a premium before the build list.

4. Chevron click affordance
Right-aligned arrow icon on every card. Animates right + turns accent-orange on hover. Communicates "this is a navigation action" — clicking takes you into the builder with mods pre-loaded. Subtitle updated to "Select a set to enter builder" to reinforce the mental model.

5. Advisor-consistent abbreviations
All mod names follow the same abbreviation rules as Advisor R3-final:
• "Compensator" → "Comp."
• "Extended" → "Ext."
• "Light" → "Lt."
• "Horizontal Grip" → "H. Grip" (mobile only, desktop keeps full name)
• "Kinetic Converter" → "K. Conv." (mobile only)
• "Angled Grip", "Stable Stock", "Padded Stock" → stay as-is
• Tier suffixes always shown (I, II, III)

6. Mod gallery layout (Advisor consistency)
Attachments are displayed as vertical icon+label columns — 36px icon on top, 10px label below — matching the Advisor R3-final gallery pattern exactly. Desktop uses 80px max-width items with 6px gap; mobile uses 70px items with 4px gap and 30px icons. This creates a unified visual language for "mod display" across the entire application. Eliminates the horizontal chip pattern that only existed in the builder.

7. Interaction model clarified
Clicking a build card = enter the builder with those mods pre-equipped. Same as the existing GoalCard behavior but with the new guide data. "Build manually" link enters builder with blank slate. This maps cleanly to the existing applyGuideBuild / resetToBuilds actions in the plan.

Graduated Tier Derivation

The left border color and build number circle are determined by the highest-rarity mod in each build. Iterate the build's mods, find max rarity using RARITY_ORDER from constants.ts, then map to a tier class:

  • Legendary → gold left border (50% opacity), gold circle
  • Epic → pink left border (45%), pink circle
  • Rare → blue left border (40%), blue circle
  • Uncommon → green left border (35%), green circle
  • Common → subtle gray border, gray circle

Use color-mix(in srgb, var(--color-rarity-X) N%, transparent) for the semi-transparent borders. The opacity decreases as rarity decreases (50→45→40→35→subtle) to reinforce the visual gradient.

Crafting Requirement Labels

Derived from highest-rarity mod, shown adjacent to range name with · separator:

  • Legendary → Desktop: "Requires Legendaries" / Mobile: "Legendaries" (gold)
  • Epic → "Requires Mod Components" / "Mod Comps" (pink)
  • Rare → "Requires Mod Components" / "Mod Comps" (blue)
  • Uncommon → "Requires Mech Components" / "Mech Comps" (green)
  • Common → "Base Materials" / "Base Materials" (text-faint)

Epic and Rare both say "Mod Components" — the color differentiates them visually.

ModGallery Shared Component

Extract ModGallery.tsx into src/components/shared/. Both Builder and Advisor use the same vertical icon+label layout. Props:

  • mods: Array<{ slot, family, tier }> — the build's mod selections
  • weaponId: string — for mod image lookup from mods.ts
  • compact?: boolean — mobile (30px icons, 9px labels, 70px items) vs desktop (36px, 10px, 80px)

The Advisor's current inline mod display should also migrate to this shared component for consistency.

Abbreviation Utility

Extract to src/utils/abbreviate.ts. Used by both Builder and Advisor:

  • "Compensator" → "Comp." (always)
  • "Extended" → "Ext." (always)
  • "Light" (as in Light Mag) → "Lt." (always)
  • "Magazine" → "Mag" (always)
  • "Shotgun" → "SG" (always)
  • "Vertical" → "Vert." (always)
  • "Horizontal Grip" → "H. Grip" (mobile only)
  • "Kinetic Converter" → "K. Conv." (mobile only)
  • Tier suffixes I, II, III always shown

Consider a compact flag that enables mobile-only abbreviations.

Interaction Model

  • Click "build manually" → dismiss build list, enter builder with blank slate
  • No persistent "active build" — once applied, mods are just regular equipped mods
  • "Clear All Attachments" → returns to build list view
